# Contributing to UTOID

Thank you for your interest in contributing. This repository is the entry point
for UTOID documentation and cross-repo coordination. It is intentionally kept
stable and release-ready.

## Scope

- Architecture and system-level documentation
- Cross-repo navigation and references
- Release notes and project-level policies

Implementation details live in the component repositories (utocoin, ipv7,
utowallet). Please open issues or pull requests in the relevant repo for code
changes.

## Before you start

- Search existing issues and pull requests to avoid duplicate work.
- For significant changes, open an issue to discuss scope first.
- Keep all documentation in English.
- Avoid placeholder notes in release documents.

## Workflow

1. **Fork and branch**
   - Create a feature branch for your changes.
2. **Make changes**
   - Keep edits focused and scoped to one topic.
   - Use consistent terminology: UTOID, A7, utocoin, IPv7, UTOWallet.
3. **Verify**
   - Ensure links are correct and files render properly.
   - Avoid time-sensitive claims unless they are stable and verifiable.
4. **Open a pull request**
   - Describe the problem, the solution, and any trade-offs.
   - Link related issues where possible.

## Documentation standards

- Use clear, concise technical English.
- Prefer stable references over speculative claims.
- Keep the system status section minimal and easy to maintain.
- Ensure filenames and headings follow consistent style.

## Review process

- Maintainers will review for accuracy, clarity, and consistency.
- Changes may be requested to align terminology or scope.
- If a change depends on implementation details, include references to the
  source repository and relevant files.
